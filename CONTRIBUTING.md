# Contributing

Thanks for helping keep this list accurate — that's the entire value of it.

## What makes a good PR

- **Fixing a stale limit or dead link.** The most valuable contribution of all. Link to the provider's own docs or pricing page as the source.
- **Adding a provider.** It must offer genuinely free API access to LLMs (permanent free tier or trial credits) — not just a web chat UI, and not "free" behind a mandatory paid plan.
- **Removing a provider** that discontinued its free tier.

## Entry format

Keep entries in the existing style:

```markdown
### Provider Name
One or two sentences: what it is and when you'd pick it over the others.
- **Card required:** Yes/No · **Best for:** the use case it wins at
- 🔗 https://official-link
```

## Ground rules

1. **Cite the provider's own docs** for any limit you add or change — no screenshots of Reddit threads.
2. **Keep it decision-focused.** This list answers "which should I pick?", not "what exists?" — a new entry should say when someone would choose it.
3. One provider or fix per PR where possible; it makes review and merge faster.
4. No affiliate or referral links.

## How

1. Fork the repo
2. Edit `README.md`
3. Open a PR with a one-line summary of what changed and your source link

Every merged PR is credited, and (bonus) earns you GitHub's Pull Shark achievement.
